# 7-Day Nextjs Web Design

Use this skill to structure a seven-day design sprint for products that will be implemented in Next.js.

## Objective

Produce one concrete deliverable per day while keeping route structure, layout structure, component boundaries, data flow, auth, and metadata visible.

## Day Targets

- Day 1: brief, users, constraints, runtime assumptions
- Day 2: route tree, app router segments, layout hierarchy
- Day 3: server versus client boundaries, rendering, data architecture
- Day 4: route-mapped wireframes and key states
- Day 5: visual system and implementation-aware component inventory
- Day 6: accessibility, metadata, auth, performance, and risk review
- Day 7: Next.js-specific developer handoff

## Constraints

- keep app router structure explicit
- include metadata and SEO planning
- identify auth and loading-state concerns
- make the final handoff actionable for Next.js engineers
