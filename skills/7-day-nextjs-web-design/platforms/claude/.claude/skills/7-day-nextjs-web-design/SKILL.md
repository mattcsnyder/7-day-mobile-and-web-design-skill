# 7-Day Nextjs Web Design

Use this skill when a user wants a seven-day design sprint for a product that will be built in Next.js.

## Goal

Produce design outputs that are implementation-aware from the start, especially around routes, layouts, component boundaries, data flow, auth, and metadata.

## Day Plan

### Day 1: Brief

- define the product problem
- identify target users and journeys
- capture technical and deployment constraints

### Day 2: Routes and Layouts

- map the route tree
- define app router segments
- plan nested layouts and protected areas

### Day 3: Rendering and Data

- compare server versus client component tradeoffs
- identify data dependencies and API boundaries
- choose a rendering direction with reasoning

### Day 4: Route-Mapped Wireframes

- create wireframes for critical routes
- map layout regions and key states

### Day 5: Visual Design and Components

- define the visual system
- identify shared components and route-specific sections
- produce high-fidelity screens

### Day 6: Validation and Metadata

- review accessibility and missing states
- identify metadata, SEO, auth, and performance risks
- close obvious implementation gaps

### Day 7: Handoff

- package route, layout, and component maps
- summarize data dependencies and open questions
- provide Next.js-focused implementation guidance

## Rules

- keep Next.js architecture visible from Day 2 onward
- make server versus client decisions explicit
- include metadata and SEO considerations
- include auth and loading-state considerations
