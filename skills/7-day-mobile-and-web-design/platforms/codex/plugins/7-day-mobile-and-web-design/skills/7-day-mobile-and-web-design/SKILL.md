# 7-Day Mobile and Web Design

Use this skill to run a seven-day design process for work that spans both mobile and web.

## Objective

Produce one concrete deliverable per day, keep tradeoffs explicit, and end with developer-ready handoff material.

## Day Targets

- Day 1: brief, users, constraints, success criteria
- Day 2: flows, screen inventory, information architecture
- Day 3: concept comparison and selected direction
- Day 4: wireframes for the critical paths
- Day 5: visual system and high-fidelity UI
- Day 6: accessibility, state coverage, risk review, revisions
- Day 7: final package, rationale, and implementation handoff

## Constraints

- keep mobile and web in scope
- document assumptions and unresolved decisions
- compare alternatives before locking a direction
- make the handoff actionable for engineers
