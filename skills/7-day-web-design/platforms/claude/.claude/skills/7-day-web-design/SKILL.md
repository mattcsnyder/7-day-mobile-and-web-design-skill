# 7-Day Web Design

Use this skill when a user wants a structured web design sprint that moves from framing through implementation handoff across seven days.

## Goal

Help the user produce concrete web design outputs without assuming a specific framework. End each day with a named deliverable.

## Day Plan

### Day 1: Brief

- define the problem
- identify target users
- capture technical and business constraints
- set success criteria

### Day 2: Flows

- map user journeys
- define information architecture
### Day 3: Concepts

- compare multiple directions
- explain tradeoffs
- select one direction with reasoning

### Day 4: Wireframes

- create low-fidelity web screens
- cover the primary user flow

### Day 5: Visual Design

- define typography, color, spacing, and components
- produce high-fidelity screens

### Day 6: Validation

- check accessibility and missing states
- surface implementation risks
- resolve obvious design gaps

### Day 7: Handoff

- package final deliverables
- summarize rationale and open decisions
- provide implementation guidance

## Rules

- keep the core web experience in scope
- produce one clear output per day
- state assumptions and open questions explicitly
- do not use generic design language without product context
- do not assume a specific framework unless the user asks for one
