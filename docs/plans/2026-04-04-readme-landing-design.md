# README Landing Design

## Goal

Restructure the root README so it works as a clearer landing page for the workspace.

The new README should lead with:
1. sponsorship
2. AgentPowers website link
3. quick links to day-by-day plans
4. instructions
5. targeted checklists

It should still explain the workspace and keep validation/export usage visible.

## Chosen Structure

The README should be organized in this order:

### Sponsorship

A short top-of-page section that makes sponsorship visible immediately.

### AgentPowers

A short section linking to `agentpowers.ai` and positioning it as the broader context or public home for the project.

### Day-by-Day Plans

Quick links to the two real skills:
- `7-day-web-design`
- `7-day-nextjs-web-design`

Each should summarize the day-by-day plan at a glance so the README acts as an index, not just a repo note.

### Instructions

Practical usage guidance for:
- generating a new skill
- editing a skill
- validating the workspace
- exporting deployable packages

### Targeted Checklists

Short checklist sections for:
- creating a new skill
- reviewing a generalized web-design skill
- reviewing a Next.js-specific skill

### Workspace Reference

Keep a compressed explanation of the repository layout so the README still functions as technical orientation for contributors.

## Constraints

- Keep the README scannable
- Avoid long prose before the top links
- Prefer direct links and short summaries
- Do not remove validation/export instructions
