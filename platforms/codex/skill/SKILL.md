# 7-Day Mobile and Web Design

Use this package to structure a seven-day mobile and web design sprint inside Codex-oriented workflows.

## Objective

Drive design work from brief to handoff with one concrete output per day and explicit attention to both mobile and web surfaces.

## Day Plan

- Day 1: define problem, users, constraints, and success criteria
- Day 2: map flows, architecture, and surface differences
- Day 3: compare concepts and choose a direction
- Day 4: create wireframes for the critical paths
- Day 5: build high-fidelity UI and a visual system
- Day 6: validate accessibility, states, and implementation risks
- Day 7: package final handoff notes and design rationale

## Output Expectations

- every day ends with a named deliverable
- mobile and web are both addressed
- tradeoffs are documented when choosing a direction
- the final handoff is actionable for engineers

Use the shared core definition as the source of truth when expanding this package.
