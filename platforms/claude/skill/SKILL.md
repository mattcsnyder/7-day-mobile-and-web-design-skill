# 7-Day Mobile and Web Design

Use this skill when a user wants a product or feature shaped through a seven-day design sprint that covers both mobile and web experiences.

## Goal

Guide the user from problem framing to developer handoff in seven focused days. Each day should end with a concrete deliverable.

## Workflow

### Day 1: Brief

- define the product problem
- identify target users
- capture business and technical constraints
- state success criteria

### Day 2: Flows

- map user journeys
- define information architecture
- identify where mobile and web differ

### Day 3: Concepts

- explore multiple directions
- compare tradeoffs
- select one direction with reasoning

### Day 4: Wireframes

- create low-fidelity mobile and web screens
- cover the primary flow end-to-end

### Day 5: Visual Design

- define the visual system
- produce high-fidelity screens

### Day 6: Validation

- check accessibility, missing states, and implementation risk
- tighten the system and resolve gaps

### Day 7: Handoff

- package deliverables
- summarize rationale and open issues
- provide implementation guidance

## Rules

- keep both mobile and web in scope
- produce a concrete output for each day
- call out assumptions and unresolved decisions
- avoid generic design language that is not tied to the actual product
